shiawase2.0に参加してきた

今日は、慶応大学三田キャンパスで開催された 第1回 shiawase 2.0 シンポジウム -HOME-というものに参加してきた。そこで感じたことを忘れないように記しておく。

動機

幸せとは？ということは常日頃から気になっていた。
本当に、最初のきっかけは多分3ヶ月くらい前、FacebookのTLにこのイベントが流れてきたのがきっかけだった。
その時は、3連休特に予定なければ行きたいな〜くらいのモチベーションだった。(運営の方々には申し訳ない…)
ただ、登壇者の中に清水ハン栄治さんがいたのは大きかった。
というのも去年禅寺で心を調える：人生と仕事に役立つマインドフルネス | Peatixというものに行ったとき、
清水ハン栄治さんが、その際の講師だったのだが、その中で行った慈悲の瞑想は僕の中で大きな印象を残していた。
また、そこから清水さんが作成したhappyという映画を知り、それを見て、普段僕が思っている違和感に何か共感のようなものを感じていたので、
清水さんにまた会ってみたいという思いは少なからず心に秘めていたのだ。
そんなこんなで、このイベントに今日は向かった。

幸せの4因子
やってみよう、ありがとう、なんとかなる、自分らしく

アドラー心理学における「幸せ」と目的論アプローチ 平本 あきお

失敗の対処法

例) 子供が父親にビールをもっていく。途中でこけてこぼしてしまった。どうする？

  1. 怒る
    完璧にしないと、だめなんだ。。。という印象を植え付けるようになり、結果的に挑戦できない人になってしまう。
  2. 親が後始末をする
    いつも誰かが後始末をしてくれるような人になる
  3. 自分で後始末をする
    こぼしたのを直して、もう一度チャレンジ。その際最初の失敗を活かす。このプロセスが結果的になんでも挑戦できる責任感のある大人を形成する
    ↑の失敗の対処法は大人にも効く。

人間は1日に6万回思考する。

リソース vs リソースフル(アンリソースフル)
リソース→ 何ができる？ということ。例えば英語が喋れる。数学ができる。など
リソースフル→心に余裕がある状態(心地よい状態)

リソースになろうとすると、上達しない。完璧にこなそうとするから。
リソースフルな状態で、完璧じゃなくとも何度も繰り返す方が後に伸びる。

人を伸ばすときは、だめと思うこともだめななかで良いところを探して褒める方が良い。
例)〜は退屈な人だよね・・・
これじゃぁ、絶対に伸びない。
退屈だけど、〜はすっごい良いよね。と言う。大事なのは「増えて欲しい」ところを伝える事。

アドラー心理学だと、しかるのもほめるのも良くないと言っている。
どうすればよいか？勇気付けをすることが大事らしい。
例)上司がいつまでも褒めたり承認していると組織は育たない。褒められた人は、上司が褒めたことに合わせるようになる。
そうすると上司が変わるとその人はうまくいかない。また、褒められた他の人が上司に認められようと頑張る。仲間のなかでの合戦が始まる。。。
良いのは、Aさんの〜Bさんがとても感謝シていたよ。と言う。横からの評価を伝えるのがとても良いと伝えていた。
